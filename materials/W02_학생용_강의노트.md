## 이번 주를 시작하기 전에

지난 주에는 변수 1개에 값 1개를 저장했다.

```python
age = 63
name = "홍길동"
```

그런데 쇼핑몰 고객이 200명이라면 어떻게 할까?

```python
age1 = 25
age2 = 30
age3 = 22
# ... 200개의 변수를 만들어야 한다 --> 불가능!
```

이 문제를 해결하는 것이 바로 **자료구조**(Data Structure)이다.
자료구조는 **여러 개의 값을 하나의 변수에 체계적으로 저장하는 방법**이다.

이번 주가 끝나면 다음 네 가지 자료구조를 자유자재로 쓸 수 있다.

| 자료구조 | 한 줄 설명 | 비유 |
|:---|:---|:---|
| **리스트**(list) | 순서 있는 수정 가능한 목록 | 장바구니 |
| **튜플**(tuple) | 순서 있는 수정 불가 목록 | 주민등록번호 |
| **딕셔너리**(dict) | 키-값 쌍의 모음 | 전화번호부 |
| **집합**(set) | 중복 없는 값의 모음 | 출석 명단 |

## 1. 리스트(List) — 파이썬의 장바구니

### 리스트란 무엇인가

**리스트**(List)는 **순서가 있고, 값을 바꿀 수 있는 여러 값의 묶음**이다.
대괄호(`[]`)로 만들고, 쉼표로 값을 구분한다.

```python
# 고객 나이 목록
ages = [25, 30, 22, 45, 35, 28]

# 고객 이름 목록
names = ["Alice", "Bob", "Charlie", "Diana"]

# 섞인 타입도 가능
mixed = [1, "hello", 3.14, True]

# 빈 리스트
empty = []
```

---

### 리스트 인덱싱과 슬라이싱

문자열과 똑같이 인덱스로 값을 꺼낼 수 있다.

```python
scores = [88, 72, 95, 64, 80]

print(scores[0])    # 88  --> 첫 번째
print(scores[-1])   # 80  --> 마지막
print(scores[1:4])  # [72, 95, 64]  --> 슬라이싱
print(scores[:3])   # [88, 72, 95]  --> 처음 3개
print(scores[::2])  # [88, 95, 80]  --> 2칸씩
```

---

### 리스트 수정 — 값을 바꾸고, 추가하고, 삭제하기

리스트는 **언제든지 수정 가능**하다. 이것이 튜플과의 핵심 차이이다.

```python
customers = ["Alice", "Bob", "Charlie"]

# 값 변경
customers[1] = "Betty"
print(customers)   # ['Alice', 'Betty', 'Charlie']

# 추가 (뒤에)
customers.append("David")
print(customers)   # ['Alice', 'Betty', 'Charlie', 'David']

# 특정 위치에 삽입
customers.insert(1, "Anna")
print(customers)   # ['Alice', 'Anna', 'Betty', 'Charlie', 'David']

# 삭제 (값으로)
customers.remove("Betty")
print(customers)   # ['Alice', 'Anna', 'Charlie', 'David']

# 삭제 (인덱스로)
del customers[0]
print(customers)   # ['Anna', 'Charlie', 'David']

# 꺼내서 삭제 (마지막 값)
last = customers.pop()
print(last)        # David
print(customers)   # ['Anna', 'Charlie']
```

> **함정 주의!** `.append()`는 반환값이 `None`이다.
> ```python
> result = customers.append("Eve")
> print(result)   # None  --> 리스트 자체가 바뀌지, 새 리스트를 반환하지 않는다
> ```

---

### 리스트의 유용한 기능들

```python
scores = [88, 72, 95, 64, 80]

# 길이
print(len(scores))         # 5

# 최대, 최솟값, 합계
print(max(scores))         # 95
print(min(scores))         # 64
print(sum(scores))         # 399

# 정렬
print(sorted(scores))      # [64, 72, 80, 88, 95]  --> 원본 유지, 새 리스트 반환
scores.sort()               # 원본 자체를 정렬 (반환값 없음!)
print(scores)               # [64, 72, 80, 88, 95]

# 뒤집기
scores.reverse()
print(scores)               # [95, 88, 80, 72, 64]

# 포함 여부 확인
print(80 in scores)         # True
print(100 in scores)        # False

# 개수 세기
nums = [1, 2, 2, 3, 2, 4]
print(nums.count(2))        # 3
```

> **함정 주의!** `sorted()`와 `.sort()`의 차이를 기억하라.
> - `sorted(리스트)` --> 원본은 그대로, **새 정렬된 리스트 반환**
> - `리스트.sort()` --> **원본 자체를 정렬**, 반환값은 `None`

---

### 리스트 복사 — 가장 중요한 함정!

리스트를 복사할 때 가장 실수가 많이 발생하는 부분이다.

```python
a = [1, 2, 3]
b = a           # 복사가 아니라 "같은 리스트를 가리키는 것"!

b.append(4)
print(a)        # [1, 2, 3, 4]  --> a도 바뀌었다! 충격!
print(b)        # [1, 2, 3, 4]
```

`b = a`는 복사가 아니다. `b`는 `a`와 **같은 리스트에 붙인 별명**이다.
진짜 복사를 하려면:

```python
a = [1, 2, 3]

# 방법 1: .copy()
b = a.copy()

# 방법 2: 슬라이싱
b = a[:]

# 방법 3: list()
b = list(a)

b.append(4)
print(a)   # [1, 2, 3]  --> 원본 유지!
print(b)   # [1, 2, 3, 4]
```

## 2. 튜플(Tuple) — 수정 불가 리스트

### 튜플이란 무엇인가

**튜플**(Tuple)은 **순서가 있지만, 한 번 만들면 수정할 수 없는** 자료구조이다.
소괄호(`()`)로 만든다.

```python
# 고객 기본 정보 (변경하면 안 되는 데이터)
customer_info = ("C001", "Alice", 25, "Female")

# 좌표 (바뀌지 않아야 함)
location = (37.5665, 126.9780)   # 서울 위도, 경도
```

---

### 튜플과 리스트의 차이

| 특징 | 리스트 | 튜플 |
|:---|:---|:---|
| 기호 | `[1, 2, 3]` | `(1, 2, 3)` |
| 수정 | 가능 | **불가** |
| 속도 | 보통 | 더 빠름 |
| 용도 | 변하는 데이터 | 변하지 않는 데이터 |

```python
t = (1, 2, 3)
print(t[0])     # 1  --> 읽기는 가능

t[0] = 99      # TypeError: 'tuple' object does not support item assignment
```

---

### 튜플 언패킹(Unpacking)

튜플의 강력한 기능이다. 여러 변수에 동시에 값을 할당할 수 있다.

```python
customer = ("Alice", 25, "Female")

name, age, gender = customer    # 언패킹!
print(name)     # Alice
print(age)      # 25
print(gender)   # Female

# 두 변수 값 교환
a, b = 10, 20
a, b = b, a
print(a, b)     # 20 10
```

## 3. 딕셔너리(Dictionary) — 파이썬의 전화번호부

### 딕셔너리란 무엇인가

**딕셔너리**(Dictionary)는 **키(key)와 값(value)의 쌍**으로 데이터를 저장하는 자료구조이다.
중괄호(`{}`)로 만들고, `키: 값` 형태로 저장한다.

실제 전화번호부처럼 "이름(키)으로 전화번호(값)를 찾는" 방식이다.

```python
# 고객 한 명의 정보
customer = {
    "id": "C001",
    "name": "Alice",
    "age": 25,
    "gender": "Female",
    "spending_score": 82
}
```

---

### 딕셔너리 값 접근

```python
print(customer["name"])           # Alice
print(customer["spending_score"]) # 82

# .get() 사용 (키가 없어도 오류 안 남)
print(customer.get("age"))        # 25
print(customer.get("email"))      # None  --> 키가 없어도 안전!
print(customer.get("email", "정보없음"))  # 정보없음  --> 기본값 지정 가능
```

> **리스트와의 차이:** 리스트는 번호(인덱스)로 찾고, 딕셔너리는 이름(키)으로 찾는다.

---

### 딕셔너리 수정, 추가, 삭제

```python
customer = {"name": "Alice", "age": 25}

# 값 수정
customer["age"] = 26
print(customer)   # {'name': 'Alice', 'age': 26}

# 키-값 추가
customer["city"] = "Seoul"
print(customer)   # {'name': 'Alice', 'age': 26, 'city': 'Seoul'}

# 삭제
del customer["city"]
print(customer)   # {'name': 'Alice', 'age': 26}

# 꺼내서 삭제
age = customer.pop("age")
print(age)        # 26
print(customer)   # {'name': 'Alice'}
```

---

### 딕셔너리 순회(Iteration)

딕셔너리의 키, 값, 또는 둘 다를 순서대로 꺼낼 수 있다.

```python
customer = {"name": "Alice", "age": 25, "gender": "Female"}

# 키만
for key in customer.keys():
    print(key)   # name, age, gender

# 값만
for value in customer.values():
    print(value)   # Alice, 25, Female

# 키와 값 모두
for key, value in customer.items():
    print(f"{key}: {value}")
# name: Alice
# age: 25
# gender: Female

# 키 존재 여부 확인
print("name" in customer)    # True
print("email" in customer)   # False
```

## 4. 집합(Set) — 중복 없는 값의 묶음

### 집합이란 무엇인가

**집합**(Set)은 **중복을 허용하지 않는 값의 모음**이다.
수학의 집합과 같은 개념이다.

```python
# 방문 고객 ID (중복 제거)
visitors = {101, 102, 103, 102, 101, 104}
print(visitors)   # {101, 102, 103, 104}  --> 중복 자동 제거!

# 리스트에서 중복 제거하는 가장 간단한 방법
ages = [25, 30, 25, 22, 30, 35, 22]
unique_ages = set(ages)
print(unique_ages)   # {25, 30, 22, 35}
```

> **주의!** 집합은 **순서가 없다**. 인덱싱(`[0]`)이 불가능하다.

---

### 집합 연산 — 교집합, 합집합, 차집합

```python
morning_customers = {101, 102, 103, 104}
evening_customers = {103, 104, 105, 106}

# 합집합 (|)
print(morning_customers | evening_customers)
# {101, 102, 103, 104, 105, 106}  --> 전체 고객

# 교집합 (&)
print(morning_customers & evening_customers)
# {103, 104}  --> 오전, 저녁 모두 방문한 고객

# 차집합 (-)
print(morning_customers - evening_customers)
# {101, 102}  --> 오전에만 방문한 고객
```

## 5. 불리언(Boolean) — 참과 거짓

### 불리언이란

**불리언**(Boolean)은 `True`(참) 또는 `False`(거짓) 둘 중 하나의 값만 갖는다.
조건을 표현할 때 사용한다.

```python
is_member = True
has_discount = False
age = 25

print(age >= 18)    # True
print(age > 30)     # False
```

---

### 비교 연산자

| 연산자 | 의미 | 예시 |
|:---|:---|:---|
| `==` | 같다 | `5 == 5` --> `True` |
| `!=` | 다르다 | `5 != 3` --> `True` |
| `>` | 크다 | `5 > 3` --> `True` |
| `<` | 작다 | `5 < 3` --> `False` |
| `>=` | 크거나 같다 | `5 >= 5` --> `True` |
| `<=` | 작거나 같다 | `5 <= 4` --> `False` |

---

### 논리 연산자

| 연산자 | 의미 | 예시 |
|:---|:---|:---|
| `and` | 둘 다 참이면 참 | `True and False` --> `False` |
| `or` | 하나라도 참이면 참 | `True or False` --> `True` |
| `not` | 반대 | `not True` --> `False` |

```python
age = 25
income = 5000

# 할인 대상: 나이가 30 미만이고 소득이 3000 이상
is_eligible = (age < 30) and (income >= 3000)
print(is_eligible)   # True

# VIP 조건: 나이가 50 이상 또는 소득이 8000 초과
is_vip = (age >= 50) or (income > 8000)
print(is_vip)   # False
```

## 6. 실전 적용 — 쇼핑몰 고객 데이터

```python
# 고객 목록 (딕셔너리를 원소로 갖는 리스트)
customers = [
    {"id": "C001", "name": "Alice",   "age": 25, "gender": "Female", "score": 82},
    {"id": "C002", "name": "Bob",     "age": 30, "gender": "Male",   "score": 45},
    {"id": "C003", "name": "Charlie", "age": 22, "gender": "Male",   "score": 72},
    {"id": "C004", "name": "Diana",   "age": 45, "gender": "Female", "score": 91},
]

# 모든 나이 추출
ages = [c["age"] for c in customers]
print(f"평균 나이: {sum(ages) / len(ages):.1f}세")

# 고득점 고객 필터링
high_scorers = [c["name"] for c in customers if c["score"] >= 70]
print(f"고득점 고객: {high_scorers}")

# 고객 ID 집합 (중복 없음)
ids = {c["id"] for c in customers}
print(f"고객 수: {len(ids)}명")
```

## 8. 자주 묻는 질문(FAQ)

**Q. 리스트와 딕셔너리 중 언제 무엇을 쓰는가?**
A. 데이터가 "순서대로 나열된 것"이면 리스트, "이름으로 찾아야 하는 것"이면 딕셔너리를 쓴다. 학생 성적 목록은 리스트, 학생 정보(이름, 나이, 성적)는 딕셔너리이다.

**Q. `sorted()`와 `.sort()`의 차이는 무엇인가?**
A. `sorted(x)`는 x를 그대로 두고 **새 정렬된 리스트를 반환**한다. `x.sort()`는 **x 자체를 정렬**하고 `None`을 반환한다.

**Q. 딕셔너리에서 없는 키를 `[]`로 접근하면 어떻게 되는가?**
A. `KeyError`가 발생한다. 안전하게 접근하려면 `.get(키)` 또는 `.get(키, 기본값)`을 사용한다.

**Q. 집합에 리스트를 원소로 넣을 수 있는가?**
A. 없다. 집합의 원소는 **변경 불가능한 값**(int, str, tuple)이어야 한다. 리스트는 변경 가능하므로 집합 원소가 될 수 없다.
