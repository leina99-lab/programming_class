## 이번 주를 시작하기 전에

지금까지 파이썬은 코드를 위에서 아래로 **한 줄씩 순서대로** 실행했다.

```python
print("첫째 줄")
print("둘째 줄")
print("셋째 줄")
```

하지만 실제 세상은 이렇지 않다.

- **"비가 오면 우산을 가져가고, 안 오면 선글라스를 가져가라"** --> 조건 분기
- **"손님이 올 때까지 문 앞에서 기다려라"** --> 반복
- **"1번부터 100번 고객 모두에게 문자를 보내라"** --> 반복

이것을 가능하게 하는 것이 **제어문**(Control Flow)이다.

이번 주가 끝나면:
- `if`/`elif`/`else`로 조건에 따라 다르게 처리한다
- `while`로 조건이 만족될 때까지 반복한다
- `for`로 목록을 순서대로 처리한다
- 함수로 반복되는 코드를 재사용한다

---

## 1. if문 — 조건에 따라 다르게 행동하기

### if문의 기본 구조

```python
if 조건:
    조건이 참일 때 실행할 코드
elif 다른 조건:
    다른 조건이 참일 때 실행할 코드
else:
    모든 조건이 거짓일 때 실행할 코드
```

> **파이썬의 들여쓰기(Indentation)는 선택이 아닌 필수이다!**
> `if` 다음 블록은 반드시 4칸 들여써야 한다. 안 쓰면 `IndentationError`가 발생한다.

---

### 실제 예시 — 게임 판매 등급 분류

```python
sales = 3200000   # 판매량 (단위: 개)

if sales >= 5000000:
    grade = "플래티넘"
elif sales >= 3000000:
    grade = "골드"
elif sales >= 1000000:
    grade = "실버"
else:
    grade = "브론즈"

print(f"판매 등급: {grade}")   # 판매 등급: 골드
```

---

### elif 순서가 중요하다

`elif`는 위에서 아래로 순서대로 검사한다. **먼저 만족되는 조건에서 멈춘다.**

```python
score = 85

# 올바른 순서
if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"   # score=85이므로 여기서 결정됨
elif score >= 70:
    grade = "C"
else:
    grade = "F"

print(grade)   # B
```

```python
# 잘못된 순서 (실수하기 쉬운 예)
if score >= 70:
    grade = "C"   # score=85이면 여기서 이미 결정되어 버림!
elif score >= 80:
    grade = "B"   # 여기 절대 도달 불가
elif score >= 90:
    grade = "A"   # 여기 절대 도달 불가

print(grade)   # C  --> 잘못된 결과!
```

---

### 중첩 if문 (Nested if)

`if`문 안에 또 `if`문을 쓸 수 있다.

```python
platform = "PS4"
sales = 1500000

if platform == "PS4":
    if sales >= 1000000:
        print("PS4 밀리언셀러!")
    else:
        print("PS4 게임 (100만 미달)")
else:
    print("다른 플랫폼 게임")
```

---

### 한 줄 if문 (삼항 연산자)

간단한 조건은 한 줄로 쓸 수 있다.

```python
sales = 2000000
label = "히트" if sales >= 1000000 else "미달"
print(label)   # 히트
```

## 2. while문 — 조건이 참인 동안 반복하기

### while문의 기본 구조

```python
while 조건:
    조건이 참인 동안 반복할 코드
```

조건이 거짓이 되는 순간 반복이 끝난다.

---

### 실제 예시 — 판매량이 목표치에 달할 때까지

```python
current_sales = 0
target = 5

print("판매 시작!")
while current_sales < target:
    current_sales += 1
    print(f"현재 판매량: {current_sales}개")

print(f"목표 달성! 최종 판매량: {current_sales}개")
```

출력:
```
판매 시작!
현재 판매량: 1개
현재 판매량: 2개
현재 판매량: 3개
현재 판매량: 4개
현재 판매량: 5개
목표 달성! 최종 판매량: 5개
```

---

### 무한루프 — 반드시 조심해야 하는 함정!

`while`문에서 조건이 절대 거짓이 되지 않으면 **무한루프**(Infinite Loop)에 빠진다.

```python
# 위험한 코드! 절대 실행하지 말 것!
i = 0
while i < 5:
    print(i)
    # i += 1 을 빠뜨리면 i는 영원히 0!
    # --> 프로그램이 멈추지 않는다
```

만약 무한루프에 빠졌다면 **Ctrl + C**를 눌러서 강제 중단한다.

---

### break와 continue

```python
# break: 루프를 즉시 탈출
i = 0
while True:   # 무한루프처럼 보이지만...
    if i >= 5:
        break   # 조건 만족시 탈출!
    print(i)
    i += 1
# 출력: 0 1 2 3 4

# continue: 현재 반복만 건너뛰고 다음으로
i = 0
while i < 10:
    i += 1
    if i % 2 == 0:
        continue   # 짝수는 건너뜀
    print(i)
# 출력: 1 3 5 7 9
```

> **continue 함정 주의!** `continue` 앞에 `i += 1`이 없으면 무한루프가 된다. `continue`는 루프의 처음으로 돌아가기 때문에 변수 증가를 건너뛰게 된다.

## 3. for문 — 목록을 하나씩 처리하기

### for문의 기본 구조

```python
for 변수 in 반복가능한_것:
    반복할 코드
```

`while`이 "조건이 참인 동안" 반복한다면, `for`는 "목록이 끝날 때까지" 반복한다.

---

### 리스트 순회

```python
genres = ["Action", "RPG", "Sports", "Puzzle"]

for genre in genres:
    print(f"장르: {genre}")

# 출력:
# 장르: Action
# 장르: RPG
# 장르: Sports
# 장르: Puzzle
```

---

### range() — 숫자 범위 생성

```python
# range(끝)  --> 0부터 끝-1까지
for i in range(5):
    print(i)   # 0 1 2 3 4

# range(시작, 끝)  --> 시작부터 끝-1까지
for i in range(1, 6):
    print(i)   # 1 2 3 4 5

# range(시작, 끝, 간격)  --> 간격만큼 건너뛰며
for i in range(0, 10, 2):
    print(i)   # 0 2 4 6 8

# 역순
for i in range(5, 0, -1):
    print(i)   # 5 4 3 2 1
```

---

### enumerate() — 인덱스와 값 동시에

```python
games = ["마리오", "젤다", "포켓몬"]

for i, game in enumerate(games):
    print(f"{i+1}번째 게임: {game}")

# 출력:
# 1번째 게임: 마리오
# 2번째 게임: 젤다
# 3번째 게임: 포켓몬
```

---

### 중첩 for문 (구구단)

```python
for i in range(2, 10):
    for j in range(1, 10):
        print(f"{i} x {j} = {i*j}", end="\t")
    print()   # 줄바꿈
```

---

### 리스트 컴프리헨션(List Comprehension) — 한 줄 for문

리스트 컴프리헨션은 `for`문을 이용해 리스트를 한 줄로 만드는 방법이다.

```python
sales_data = [120, 350, 80, 220, 410, 55]

# 일반 for문
million_games = []
for s in sales_data:
    if s >= 200:
        million_games.append(s)

# 리스트 컴프리헨션 (한 줄!)
million_games = [s for s in sales_data if s >= 200]
print(million_games)   # [350, 220, 410]

# 변환까지 포함
doubled = [s * 2 for s in sales_data]
print(doubled)   # [240, 700, 160, 440, 820, 110]
```

## 4. 함수(Function) — 재사용 가능한 코드 묶음

### 함수가 필요한 이유

같은 코드를 여러 번 반복하면 파일이 길어지고 수정하기 어려워진다.

```python
# 나쁜 예 (반복되는 코드)
print("환자 Alice의 등급 계산 중...")
if 82 >= 80:
    print("VIP 고객")

print("환자 Bob의 등급 계산 중...")
if 45 >= 80:
    print("VIP 고객")
else:
    print("일반 고객")
```

함수를 쓰면 코드를 한 번만 작성하고 여러 번 사용할 수 있다.

---

### 함수 정의와 호출

```python
# 함수 정의
def get_customer_grade(score):
    """점수를 입력받아 등급을 반환하는 함수"""
    if score >= 80:
        return "VIP"
    elif score >= 50:
        return "일반"
    else:
        return "이탈위험"

# 함수 호출
grade1 = get_customer_grade(82)
grade2 = get_customer_grade(45)
grade3 = get_customer_grade(60)

print(grade1)   # VIP
print(grade2)   # 이탈위험
print(grade3)   # 일반
```

---

### 매개변수와 기본값

```python
def greet(name, message="안녕하세요"):
    """기본값이 있는 매개변수"""
    print(f"{name}님, {message}!")

greet("Alice")            # Alice님, 안녕하세요!
greet("Bob", "반갑습니다") # Bob님, 반갑습니다!
```

---

### 여러 값 반환

```python
def analyze_sales(sales_list):
    """판매 데이터 분석 결과를 여러 값으로 반환"""
    total = sum(sales_list)
    average = total / len(sales_list)
    best = max(sales_list)
    worst = min(sales_list)
    return total, average, best, worst

data = [120, 350, 80, 220, 410]
total, avg, best, worst = analyze_sales(data)

print(f"총합: {total}")
print(f"평균: {avg:.1f}")
print(f"최고: {best}")
print(f"최저: {worst}")
```

---

### 가변 인수 (*args, **kwargs)

```python
# *args: 개수가 정해지지 않은 위치 인수
def sum_all(*args):
    total = 0
    for n in args:
        total += n
    return total

print(sum_all(1, 2, 3))        # 6
print(sum_all(10, 20, 30, 40)) # 100

# **kwargs: 개수가 정해지지 않은 키워드 인수
def print_info(**kwargs):
    for key, value in kwargs.items():
        print(f"{key}: {value}")

print_info(name="Alice", age=25, city="Seoul")
# name: Alice
# age: 25
# city: Seoul
```

## 5. 실전 적용 — 게임 판매 데이터 분석

```python
# 게임 판매 데이터
games = [
    {"title": "Grand Theft Auto V",  "platform": "PS4",  "sales": 6000000, "genre": "Action"},
    {"title": "FIFA 22",             "platform": "PS4",  "sales": 3200000, "genre": "Sports"},
    {"title": "Minecraft",           "platform": "PC",   "sales": 2400000, "genre": "Sandbox"},
    {"title": "Call of Duty",        "platform": "Xbox", "sales": 1800000, "genre": "Action"},
    {"title": "Animal Crossing",     "platform": "Switch","sales": 900000, "genre": "Simulation"},
]

def classify_game(sales):
    """판매량에 따른 등급 분류"""
    if sales >= 5000000:
        return "플래티넘"
    elif sales >= 2000000:
        return "골드"
    elif sales >= 1000000:
        return "실버"
    else:
        return "브론즈"

# 모든 게임 등급 출력
print(f"{'제목':<30} {'플랫폼':<8} {'판매량':>10} {'등급'}")
print("-" * 60)
for g in games:
    grade = classify_game(g["sales"])
    print(f"{g['title']:<30} {g['platform']:<8} {g['sales']:>10,} {grade}")

# 플래티넘 게임만 필터링
platinum = [g["title"] for g in games if classify_game(g["sales"]) == "플래티넘"]
print(f"\n플래티넘 게임: {platinum}")
```

## 6. 핵심 정리

### 제어문 요약

| 구문 | 용도 | 종료 조건 |
|:---|:---|:---|
| `if/elif/else` | 조건 분기 | 해당 없음 (한 번 실행) |
| `while` | 조건 반복 | 조건이 거짓이 될 때 |
| `for` | 목록 반복 | 목록이 끝날 때 |

### 함수 핵심

```python
def 함수이름(매개변수1, 매개변수2=기본값):
    """독스트링 (함수 설명)"""
    # 실행할 코드
    return 반환값
```

### 자주 쓰는 패턴

```python
# 조건 필터링
result = [x for x in data if 조건]

# 값 변환
result = [변환(x) for x in data]

# 인덱스와 함께
for i, x in enumerate(data):
    ...

# 딕셔너리 순회
for key, value in dict.items():
    ...
```

## 7. 자주 묻는 질문(FAQ)

**Q. `while`과 `for` 중 언제 무엇을 써야 하는가?**
A. 반복 횟수를 미리 알 때는 `for`, 조건이 만족될 때까지 반복할 때는 `while`을 쓴다. 리스트를 순서대로 처리할 때는 항상 `for`가 더 자연스럽다.

**Q. `break`와 `continue`의 차이는?**
A. `break`는 루프 전체를 즉시 탈출한다. `continue`는 현재 반복만 건너뛰고 다음 반복으로 진행한다.

**Q. 함수에서 `return`이 없으면 어떻게 되는가?**
A. 파이썬이 자동으로 `None`을 반환한다.

**Q. 리스트 컴프리헨션은 언제 쓰고 언제 일반 `for`문을 쓰는가?**
A. 코드가 한 줄에 표현될 만큼 단순할 때는 컴프리헨션, 복잡한 로직이 필요할 때는 일반 `for`문이 읽기 더 좋다.
