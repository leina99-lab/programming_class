## 3. 탐색적 데이터 분석(EDA) 전체 흐름

**EDA**(Exploratory Data Analysis)는 데이터를 처음 만났을 때 수행하는 탐색 과정이다.

```
1단계: 데이터 불러오기 & 기본 확인
     --> df.shape, df.info(), df.head()

2단계: 결측값 확인 & 처리
     --> df.isnull().sum(), df.fillna(), df.dropna()

3단계: 이상치 확인 & 처리
     --> df.describe(), 박스플롯, 히스토그램

4단계: 분포 시각화
     --> 히스토그램, 박스플롯 (각 수치형 변수)

5단계: 관계 분석
     --> 상관관계 히트맵, 산점도, 페어플롯

6단계: 그룹별 비교
     --> groupby + 막대그래프
```

## 4. 함수 심화 — 람다와 고차 함수

### 람다(Lambda) 함수 — 이름 없는 한 줄 함수

```python
# 일반 함수
def double(x):
    return x * 2

# 람다 함수 (같은 기능, 한 줄)
double_lambda = lambda x: x * 2

print(double(5))        # 10
print(double_lambda(5)) # 10

# 람다의 진짜 용도: 다른 함수의 인수로 사용
prices = [150, 80, 220, 65, 300]

# sorted()의 key 인수에 람다 사용
rooms = [
    {"type": "Entire home", "price": 200},
    {"type": "Private room", "price": 80},
    {"type": "Shared room", "price": 40},
]

# 가격순 정렬
sorted_rooms = sorted(rooms, key=lambda r: r["price"])
for r in sorted_rooms:
    print(f"{r['type']}: ${r['price']}")
```

---

### map(), filter(), sorted() — 고차 함수

```python
prices = [150, 80, 220, 65, 300, 120]

# map(): 모든 원소에 함수 적용
doubled = list(map(lambda p: p * 2, prices))
print(doubled)   # [300, 160, 440, 130, 600, 240]

# filter(): 조건에 맞는 원소만
expensive = list(filter(lambda p: p >= 150, prices))
print(expensive)   # [150, 220, 300]

# sorted() with key
sorted_prices = sorted(prices, reverse=True)
print(sorted_prices)   # [300, 220, 150, 120, 80, 65]
```

## 5. 예외처리(Exception Handling) — 오류를 안전하게 다루기

### 왜 예외처리가 필요한가

프로그램은 예상치 못한 상황에서 갑자기 멈출 수 있다.

```python
# 이 코드는 값이 "없음"일 때 오류가 발생한다
price_str = "없음"
price = int(price_str)   # ValueError: invalid literal for int() with base 10: '없음'
print(f"가격: {price}")   # 여기까지 도달하지 못한다
```

예외처리를 쓰면 오류가 발생해도 프로그램이 멈추지 않는다.

---

### try / except 기본 구조

```python
try:
    # 오류가 발생할 수 있는 코드
    result = 10 / 0
except ZeroDivisionError:
    # 해당 오류 발생시 실행
    print("0으로 나눌 수 없습니다.")
```

---

### 여러 예외 처리하기

```python
def safe_convert_price(price_str):
    """문자열을 숫자로 안전하게 변환"""
    try:
        price = float(price_str)
        if price < 0:
            raise ValueError("가격은 음수일 수 없습니다.")
        return price
    except ValueError as e:
        print(f"변환 오류: {e}")
        return None
    except TypeError as e:
        print(f"타입 오류: {e}")
        return None
    finally:
        print("변환 시도 완료")   # 오류 유무에 상관없이 항상 실행

print(safe_convert_price("150"))    # 150.0
print(safe_convert_price("없음"))   # None
print(safe_convert_price(-50))      # None (음수)
```

---

### 자주 만나는 오류들

| 오류 | 원인 | 예시 |
|:---|:---|:---|
| `NameError` | 정의되지 않은 변수 | `print(x)` (x 미정의) |
| `TypeError` | 잘못된 타입 연산 | `"5" + 5` |
| `ValueError` | 올바르지 않은 값 | `int("abc")` |
| `IndexError` | 범위 벗어난 인덱스 | `lst[10]` (lst 길이 5) |
| `KeyError` | 없는 딕셔너리 키 | `d["없는키"]` |
| `ZeroDivisionError` | 0으로 나눔 | `10 / 0` |
| `FileNotFoundError` | 파일 없음 | `open("없는파일.csv")` |

## 7. 실전 프로젝트 — 에어비앤비 완전 분석

```python
import pandas as pd
import numpy as np
import matplotlib.pyplot as plt
import seaborn as sns
import warnings
warnings.filterwarnings("ignore")

def load_and_clean(filepath):
    """데이터 로드 및 기본 정제"""
    df = pd.read_csv(filepath)
    df["reviews_per_month"] = df["reviews_per_month"].fillna(0)
    df = df[df["price"] > 0]
    df = df[df["price"] < df["price"].quantile(0.99)]
    return df

def eda_summary(df):
    """EDA 요약 출력"""
    print("="*50)
    print("뉴욕 에어비앤비 EDA 요약")
    print("="*50)
    print(f"총 숙소: {len(df):,}개")
    print(f"결측 비율: {df.isnull().sum().sum() / (len(df) * len(df.columns)) * 100:.1f}%")
    print()

    print("[ 자치구별 통계 ]")
    stats = df.groupby("neighbourhood_group")["price"].agg(["count", "mean", "median"])
    stats.columns = ["숙소 수", "평균가", "중간값"]
    print(stats.round(0).to_string())
    print()

    print("[ 방 유형별 평균가 ]")
    print(df.groupby("room_type")["price"].mean().round(0).sort_values(ascending=False))

def visualize_summary(df):
    """핵심 시각화 4종"""
    fig, axes = plt.subplots(2, 2, figsize=(14, 10))
    fig.suptitle("뉴욕 에어비앤비 핵심 분석", fontsize=16, fontweight="bold")

    # 1. 자치구별 숙소 수
    area_cnt = df["neighbourhood_group"].value_counts()
    axes[0, 0].bar(area_cnt.index, area_cnt.values, color=sns.color_palette("Set2", 5))
    axes[0, 0].set_title("자치구별 숙소 수")
    for i, v in enumerate(area_cnt.values):
        axes[0, 0].text(i, v + 50, f"{v:,}", ha="center")

    # 2. 가격 분포
    axes[0, 1].hist(df["price"], bins=60, color="steelblue", edgecolor="white")
    axes[0, 1].axvline(df["price"].median(), color="red", linestyle="--",
                       label=f"중간값: ${df['price'].median():.0f}")
    axes[0, 1].legend()
    axes[0, 1].set_title("가격 분포")

    # 3. 방 유형 파이차트
    room_cnt = df["room_type"].value_counts()
    axes[1, 0].pie(room_cnt.values, labels=room_cnt.index, autopct="%1.1f%%",
                   colors=sns.color_palette("pastel"))
    axes[1, 0].set_title("방 유형 비율")

    # 4. 자치구별 박스플롯
    sns.boxplot(data=df, x="neighbourhood_group", y="price",
                palette="Set3", ax=axes[1, 1])
    axes[1, 1].set_title("자치구별 가격 분포")
    axes[1, 1].set_xlabel("자치구")

    plt.tight_layout()
    plt.show()

# 실행
df = load_and_clean("AB_NYC_2019.csv")
eda_summary(df)
visualize_summary(df)
```

## 9. 자주 묻는 질문(FAQ)

**Q. `plt.show()`는 항상 써야 하는가?**
A. Jupyter Notebook에서는 없어도 그래프가 표시되지만, 일반 파이썬 스크립트에서는 반드시 써야 한다. 습관적으로 쓰는 것이 좋다.

**Q. 그래프가 한글로 표시되지 않는다면?**
A. 운영체제에 따라 폰트를 설정해야 한다. Mac은 `'AppleGothic'`, Windows는 `'Malgun Gothic'`, Linux는 `'NanumGothic'`을 `matplotlib.rcParams['font.family']`에 설정한다.

**Q. `try/except`에서 Exception 하나로 모든 오류를 잡으면 안 되는가?**
A. 가능하지만 권장하지 않는다. `except Exception`은 어떤 오류인지 특정하기 어려워 디버깅이 힘들다. 예상되는 오류 유형을 구체적으로 명시하는 것이 좋다.

**Q. 람다 함수는 언제 쓰고 일반 함수는 언제 쓰는가?**
A. 람다는 한 번만 쓰이는 간단한 함수에 적합하다. 재사용하거나 복잡한 로직이 있으면 `def`로 이름 있는 함수를 만드는 것이 가독성이 좋다.
